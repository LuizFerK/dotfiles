# Dotfiles

This is a NixOS system configuration managed with Nix Flakes and home-manager. Everything is declarative: the system, user environment, installed packages, program settings, and dotfiles are all defined in this repository.

## Structure

`flake.nix` is the entry point. It declares all external inputs (nixpkgs stable 26.05, nixpkgs-unstable, home-manager, disko, caelestia-shell, expert) and wires them into the NixOS output.

`nixos/` holds system-level configuration:

- `configuration.nix` — services, hardware, users, display server, audio, networking
- `hardware-configuration.nix` — auto-generated by NixOS at install time; do not edit manually
- `disko.nix` — declarative disk partitioning (Btrfs on LVM over NVMe); only relevant at install time
- `overlays.nix` — exposes `pkgs.unstable` from nixpkgs-unstable for selective newer packages

`home/` holds the user environment managed by home-manager:

- `home.nix` — entry point; imports all modules and configures nixpkgs
- `modules/` — one file per tool or concern (git, fish, kitty, hyprland, zed, fonts, lsp, etc.)

`configs/` holds static dotfiles that are version-controlled and symlinked into the home directory via `home/modules/files.nix`. This includes the SSH config, Starship theme, Vesktop CSS theme, and the Claude Code context file.

`assets/` holds wallpapers and GIFs used by the Caelestia shell.

## How to Apply Changes

To rebuild and switch the system after any change:

```
sudo nixos-rebuild switch --flake ~/.dotfiles
```

To update all flake inputs:

```
sudo nix flake update --flake ~/.dotfiles
```

## Key Patterns

New programs belong in `home/modules/` as their own file, then imported in `home/home.nix`. Do not add program configuration directly to `home.nix` or `configuration.nix` unless it is truly system-level (services, kernel, hardware).

When a newer package version is needed from unstable, reference it as `pkgs.unstable.<name>` rather than changing the base nixpkgs input.

Static config files (non-Nix dotfiles) go in `configs/` and are linked to the home directory in `home/modules/files.nix`, not committed directly to `~`.

## Language Servers

The LSP setup lives in `home/modules/lsp.nix`. Elixir uses Expert (from the `expert` flake input). Nix uses nil and nixd.

## Desktop Stack

The window manager is Hyprland (Wayland). The display manager is SDDM with KDE Plasma 6 as fallback. The status bar and shell framework is Caelestia Shell, configured in `home/modules/caelestia.nix`. The terminal is Kitty. The primary editor is Zed.
