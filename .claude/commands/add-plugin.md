Add a Claude Code plugin to this NixOS dotfiles setup. Configures the plugin persistently across rebuilds.

## What you need from the user

If not provided, ask for:
- Plugin name and GitHub repo (e.g. `github:owner/repo`)
- Which subdirectory inside the repo is the plugin root (if not the repo root — check for a `SKILL.md` or `plugin.json` at root vs a subdirectory)

## Steps

### 1. Add flake input

In `flake.nix`, add the repo as a flake input under `inputs`. Use `flake = false` since plugin repos are not Nix flakes:

```nix
plugin-name = {
  url = "github:owner/repo";
  flake = false;
};
```

Also add `plugin-name` to the `outputs` function arguments so it's accessible as `inputs.plugin-name`.

### 2. Add to pluginDirs in claude.nix

In `home/modules/claude.nix`, add the plugin directory to the `pluginDirs` list:

```nix
"${inputs.plugin-name}"              # if plugin is at repo root
"${inputs.plugin-name}/plugins/name" # if plugin lives in a subdirectory
```

The `pluginFlags` derivation automatically turns this into `--plugin-dir` arguments passed to every `claude` invocation — no other change needed.

### 3. Update configs/CLAUDE.md

Add an entry to the Plugins section in `configs/CLAUDE.md` (the global CLAUDE.md, symlinked to `~/.claude/CLAUDE.md`) describing what the plugin does and when to use it. Be specific — the agent reads this section to know what tools are available.

### 4. Rebuild

```
sudo nixos-rebuild switch --flake ~/.dotfiles
```

## Notes

- Do not modify `settings.json` directly — it is generated by Nix from the `settings` derivation in `claude.nix`. If the plugin requires entries in `settings.json` (e.g. `mcpServers`), add them to the `builtins.toJSON { ... }` block in that derivation.
- If the plugin requires a WebFetch domain permission, add it to the `permissions.allow` list in `.claude/settings.local.json`.
- After rebuild, the plugin is active in new `claude` sessions automatically — no manual `claude` restart needed if already in a session with `--plugin-dir` flags.
